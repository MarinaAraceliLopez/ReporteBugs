Reporte de Bugs en Clockify

[Informes] Error al exportar informe de horas trabajadas: archivo PDF vacío al no haber datos disponibles.

Descripción:

Se reporta que al intentar generar un informe de horas trabajadas sin datos, el sistema genera un archivo PDF que no contiene información, lo que puede generar confusión para el usuario.

Precondiciones:

1- El usuario ha registrado previamente el correo electrónico en el sistema </EMAIL>.
2- El usuario debe estar en la página principal de Clockify </URL>.
3- El usuario se encuentra en la sección "Espacio de trabajo".
5- El sistema no cuenta con datos de entradas de tiempo registradas.
6- La configuración del idioma del sistema debe estar en "Español".

Pasos:

1- El usuario hace click en la sección "Informes".
2- El usuario hace click en el dropdown "Proyectos" de filtros.
3- El usuario hace click la casilla "Sin proyectos".
4- El usuario hace click en en el botón " Aplicar filtro".
6- El usuario hace click en el dropdown "Exportar".
7- El usuario hace click en el botón "Guardar como PDF.

Resultado Esperado:

El sistema debería mostrar un mensaje claro indicando que no hay datos para exportar y no generar un archivo PDF vacío.

Resultado Obtenido:

El sistema muestra un mensaje de "Exportando informe PDF", genera un archivo PDF, pero el archivo no contiene datos de horas trabajadas.

Detalles del Problema:

Prioridad: Alta.

Severidad: Alta.

Entorno:

Escritorio: Windows 10.

Ambiente: Producción.

Archivos Adjuntos:


Justificación

Prioridad Alta: Este problema afecta directamente a la funcionalidad de un reporte esencial que los usuarios necesitan para su gestión diaria. Solucionarlo es crucial para evitar malentendidos y mantener la confianza del usuario en la herramienta.


Severidad Alta: La falta de información en un reporte: puede causar confusión significativa para los usuarios y afectar la credibilidad del sistema, especialmente en un entorno profesional donde la precisión de los reportes es crucial.

[Proyectos] Discrepancia en la traducción del nombre del proyecto en el mensaje de éxito y en la lista de proyectos.

Descripción:

Al intentar crear un proyecto llamado "NDA" en la sección "Proyectos" de Clockify, el sistema muestra un mensaje de éxito que indica "Se ha creado el proyecto NDA", pero en la lista de proyectos disponibles se muestra como "Acuerdo de Confidencialidad". Esta diferencia se debe a un problema en la traducción, lo que genera confusión en los usuarios.

Precondiciones:

1- El usuario ha registrado previamente el correo electrónico en el sistema </EMAIL>.
2- El usuario debe estar en la página principal de Clockify </URL>.
3- El usuario se encuentra en la sección "Espacio de trabajo".
4- La configuración del idioma del sistema debe estar en "Español".

Pasos:

1- El usuario hace click en la sección "Proyectos"
2- El usuario hace click en el botón de "Crear nuevo proyecto"
3- El usuario ingresa el nombre del proyecto "NDA"
4- El usuario hace click en el botón "Crear".

Resultado Esperado:

El sistema debe mostrar un mensaje de éxito "Se ha creado el proyecto NDA".
El proyecto debe aparecer en la lista de proyectos disponibles con el nombre "NDA".

Resultado Obtenido:

El sistema muestra el mensaje de éxito correcto.
Sin embargo, el proyecto aparece en la lista de proyectos como "Acuerdo de Confidencialidad".

Detalles del Problema:

Prioridad: Alta.

Severidad: Media.

Entorno:

Escritorio: Windows 10

Ambiente: Producción.

Archivos Adjuntos:



Justificación

Prioridad Alta: La traducción incorrecta de las siglas "NDA" al inglés Aunque este problema genera una experiencia de usuario inconveniente, no impide el uso de la aplicación ni afecta sus funciones principales. Los usuarios aún pueden acceder a la aplicación y realizar las tareas necesarias, aunque con un retraso temporal.



Severidad Media: Aunque la funcionalidad principal del sistema (la creación de proyectos):sigue operando correctamente, la traducción incorrecta de las siglas afecta la experiencia del usuario y la precisión de la información presentada. Este problema no impide que los usuarios utilicen la aplicación, pero sí afecta la usabilidad y la confianza en la exactitud de los datos presentados. Dado que es un error de presentación y no un fallo crítico en la funcionalidad, se clasifica como severidad media.

[Contador de tiempo] El mensaje de confirmación no desaparece correctamente y se repite como un bucle.

Descripción:

Al ejecutar acciones consecutivas en la sección "Contador de Tiempo" de la App Clockify, el mensaje de confirmación "Entrada de tiempo creada con éxito" no desaparece como se esperaba y se repite junto con el mensaje "No se puede crear una entrada de tiempo mientras otra está en marcha". Esto afecta la experiencia del usuario al generar confusión y dificulta la comprensión de las acciones realizadas.

Precondiciones:

1- El usuario ha registrado previamente el correo electrónico en el sistema </EMAIL>.
2- El usuario debe estar en la página principal de Clockify </URL>.
3- El usuario se encuentra en la sección "Espacio de trabajo".
5- El sistema no cuenta con datos de entradas de tiempo registradas.
-6 La configuración del idioma del sistema debe estar en "Español".

Pasos:

1- El usuario hace click en la sección "Contador de tiempo".
2- El usuario hace click en el botón "Play" para iniciar el proyecto "testing functional".
3- El usuario hace click en el botón "Play" inmediatamente después repetidamente.

Resultado Esperado:

El sistema debería mostrar el mensaje "Entrada de tiempo creada con éxito", que debería desaparecer después de [tiempo esperado] segundos. Luego, debería mostrarse el mensaje "No se puede crear una entrada de tiempo mientras otra está en marcha", sin que haya duplicación del mensaje de confirmación.

Resultado Obtenido

El mensaje "Entrada de tiempo creada con éxito" aparece, pero no desaparece y se repite en bucle con el mensaje "No se puede crear una entrada de tiempo mientras otra está en marcha".

Detalles del Problema:

Prioridad: Baja.

Severidad: Media.

Entorno:

Dispositivo: Móvil.

OS: Andriod.

Ambiente: Producción.

Archivos Adjuntos:

Justificación

Prioridad Baja: El bug afecta una funcionalidad crítica de la aplicación: la gestión del tiempo y la creación de entradas. Los usuarios dependen de la capacidad de la aplicación para mostrar información clara y precisa al realizar acciones consecutivas. La confusión causada por los mensajes que no desaparecen puede llevar a errores en el seguimiento del tiempo y en la facturación, lo que impacta negativamente en la eficiencia del trabajo y la experiencia del usuario. Debido a esto, es fundamental abordar este problema de inmediato para asegurar que los usuarios puedan utilizar la aplicación sin inconvenientes.



Severidad Media: El bug afecta la experiencia del usuario y puede causar confusión: no impide por completo el uso de la aplicación. El usuario todavía puede crear entradas de tiempo, aunque de manera incorrecta o confusa. Esto significa que, a pesar de que es un problema que necesita ser resuelto, no es un fallo crítico que bloquee completamente el funcionamiento de la aplicación. Por lo tanto, la corrección es importante, pero no es tan urgente como para ser considerada una falla catastrófica.
